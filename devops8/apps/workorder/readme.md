## 工单核心逻辑:

工单申请:

    1.工单内容,标题,指派给谁,提交文件

    2.工单类型

    3.工单处理指派人

运维审批:
    待处理工单列表:

        开发人员: 只能看到自己提交工单,可以更新删除自己的工单
        运维人员: 可以看到所有未处理的工单,可以执行取消工单

处理工单:
    待处理工单列表,

    状态:
        开发人员:我正工单现在到哪儿了?
        运维人员:这个工单到底有没有人在处理?我要不要接过来?

结果回溯:
    历史工单列表,

    状态:
        开发人员:只能看到自己提交工单
        运维人员:可以看到所有工单信息
